🧠 Intuition (simple but accurate)

Think of an event loop like a smart scheduler:

It keeps a list of tasks (coroutines)
It runs a task until it hits a waiting point (await)
While that task is waiting (e.g., API call, DB query), it switches to another task
It keeps cycling until everything is done

👉 So instead of blocking, it juggles tasks efficiently


⚙️ Where it lives

Python provides event loops via:

asyncio (standard library) → main async framework

🔁 How it works internally (important)

Event loop continuously does:

Pick a task from queue
Execute until await
Pause it
Move to next ready task
Resume paused tasks when their I/O completes

This is called cooperative multitasking
→ Tasks voluntarily give up control (await)

📘 Chapter 1 — Why Async Exists (Mental Model)
🔥 Core Problem
Normal Python code is blocking.

import time

def fetch_data():
    time.sleep(2)  # simulate API call
    return "data"

def main():
    print(fetch_data())
    print(fetch_data())
    print(fetch_data())

main()
What happens?
Call 1 → waits 2 sec

Call 2 → waits 2 sec

Call 3 → waits 2 sec

👉 Total = 6 seconds

This is sequential blocking execution.

🧠 Key Insight
Most backend systems are I/O bound, not CPU bound.

I/O-bound = waiting
API calls

DB queries

File reads

Network

👉 CPU is idle, just waiting.

⚔️ Sync vs Async Thinking
Sync (Blocking)
"Do one thing at a time, wait until it's done."

Async (Non-blocking)
"Start work, don’t wait, handle other work while waiting."

🏎️ Real-World Analogy
Sync waiter:
Takes order

Goes to kitchen

Waits there

Comes back

Async waiter:
Takes order

Sends to kitchen

Serves other tables meanwhile

👉 Same time, more work done

⚙️ Async Version
import asyncio

async def fetch_data():
    await asyncio.sleep(2)  # non-blocking wait
    return "data"

async def main():
    results = await asyncio.gather(
        fetch_data(),
        fetch_data(),
        fetch_data()
    )
    print(results)

asyncio.run(main())
What happens now?
All 3 start together

Wait happens in parallel (concurrently)

Total time ≈ 2 seconds

🧩 Important Distinction
❌ Async ≠ Parallelism
Concept	Meaning
Concurrency	Handling multiple tasks at once
Parallelism	Running on multiple CPUs simultaneously
👉 Async = Concurrency (single thread, smart scheduling)

🧨 Critical Rule (Most People Miss This)
Async only helps when:

Tasks are waiting (I/O)

Async does NOT help when:

Heavy CPU work (e.g., image processing)

🧪 Example Comparison
Sync Timing
# 3 tasks × 2 sec each = 6 sec
Async Timing
# All overlap → ~2 sec total
👉 That’s a 3x improvement without extra hardware

🔥 Issue 1 — Local vs Global Variable (Core Bug)
Inside the function:

total = total + 2
Python thinks:

“total is a local variable (because you're assigning to it)”

But before assigning, you're trying to read it:

total + 2
👉 So Python throws:

UnboundLocalError: local variable 'total' referenced before assignment
🧠 Why this happens (important mental model)
Python rule:

If a variable is assigned inside a function, it's treated as local

Unless explicitly marked global

✅ Fix 1 — Use global (not recommended in real systems)
import time

total = 0 

def fake_api_call(id):
    global total
    time.sleep(2)
    total = total + 2
    return total

def main():
    print(fake_api_call(1))
    print(fake_api_call(2))
    print(fake_api_call(3))

main()
⚠️ Why this is bad design
Hidden state

Not thread-safe / async-safe

Hard to debug

Violates clean architecture principles

✅ Fix 2 — Proper Functional Approach (Recommended)
Make it stateless:

import time

def fake_api_call(id):
    time.sleep(2)
    return 2

def main():
    total = 0
    total += fake_api_call(1)
    total += fake_api_call(2)
    total += fake_api_call(3)

    print(total)

main()

No shared mutable state

Predictable

Testable

📘 Chapter 2 — Coroutines & async / await
🧠 First Principle
👉 async def does NOT execute the function

    return "data"

print(result)
Output:
<coroutine object fetch at 0x...>
👉 That is NOT the result

🔍 What is a Coroutine?
A coroutine is:

A pausable function that can be resumed later by the event loop

Think:

Normal function → runs immediately

Coroutine → returns a plan of execution

⚙️ Execution Flow
async def fetch():
    return "data"
Steps:

Call fetch() → returns coroutine object

Nothing runs yet ❗

Event loop runs it

Now it executes

🔥 Enter await
async def fetch():
    return "data"

async def main():
    result = await fetch()
    print(result)


What await does:
Tells Python:

“Pause here until this coroutine finishes”

this is the invalid usage 

def main():
    result = await fetch()  # ❌ ERROR\

await only works inside async def


🏗️ Proper Entry Point

import asyncio
async def main():
    results = await fetch()
    print(result)

asyncio.run(main())

👉 asyncio.run():

Creates event loop
Runs coroutine
Cleans up
